Class QUERY_CLASS

Query class object that holds information about the current query being built.
QUERY_CLASS:New(tableName, queryType)
This is called internally, use the corresponding methods. Creates a new query object.

Parameters:

  • tableName string The name of the table to manipulate.
  • queryType string The type of query to start building. (i.e CREATE, UPDATE, INSERT, etc.)

Parameters:

Returns:

    string The escaped string.

serverguard.mysql:Connect(host, username, password, database[, port][, socket], flags)
Connects to the SQL database with the given information.

Parameters:

  • host string The host name.
  • username string The username.
  • password string The password.
  • database string The name of the database.
  • port number The port to use. Defaults to 3306. (optional)
  • socket string The unix socket to use. Defaults to none. (optional)
  • flags number The query flags to use. Usually these aren't needed.
serverguard.mysql:RawQuery(query, callback, flags)
Sends a query to the database without any preprocessing. You should not use this unless you know what you're doing.

Parameters:

  • query string The query to run.
  • callback function The function to call when the query finishes.
  • flags number The flags to set on the query. Usually this isn't needed.
serverguard.mysql:Queue(queryString, callback)
Adds a query to the queue to be executed once previously pending ones have finished.
